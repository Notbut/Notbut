面向对象编程(基础)

类与对象

类与对象的引出

假如有多只猫，它们有年龄、性别、颜色。

  1. 如果用变量储存信息，这不利于添加某只猫的新特征，比如体重，如果猫多的时候不利于数据管理，效率低下。
  2. 如果用数组储存信息，数据类型不能得到很好的体现，例如String[] cat = {“小白”,”20”,”花色”}、而且如果想得到猫的信息，只能通过下标获取、不能体现猫的行为

快速入门

对现在内存中存在的形式

注意：

  1. JVM根据数据类型不同来存放不同的空间，字符串是引用类型，按照JVM的规则，他会把字符串放在方法区的常量池的空间
  2. 在创建Cat cat = new Cat()对象时，它会把类的信息加载到方法区

属性和成员变量

  1. 成员变量 = 属性 = field(字段)
  2. 属性是类的一个组成部分，一般是基本数据类型，也可以是引用类型(对象，数组)。比如前面定义Cat类的int就是属性

注意事项和细节

  1. 属性的定义语法和变量是一样的，示例：访问修饰符 属性类型 属性名;
  2. 属性的定义可以是任意类型，包含基本数据类型或引用类型
  3. 属性如果不赋值，有默认值，规则和数组一致

创建对象的两种形式：

  1. 先声明在创建：Cat cat; cat = new Cat();
  2. 直接创建：Cat cat = new Cat();

访问属性：

对象名.属性名;

类与对象的内存分配机制

Java内存的结构分析：

  1. 栈：一般存放基本数据类型(局部变量)
  2. 堆：存放对象(Cat cat，数组)
  3. 方法区：常量池(常量，比如字符串)，类加载信息

成员方法

基本介绍

在某些情况下，我们需要定义成员方法。比如人类：除了一些属性外(年龄，姓名)，我们还有一些行为：跑步，学习。这时就要用成员方法才能完成。(就是C语言中的函数)

成员方法的定义(也叫 方法)

注意事项和细节：

  1. 方法在类中
  2. 访问修饰符(作用是控制方法的范围)，如果不写默认访问，有四种（public protected private 默认）
  3. 最多一个返回值，返回的类型不受限制（数组
  4. 如果有返回类型，必须有return
  5. 如果方法返回类型是void，只写return或不写

快速入门

方法的调用机制原理

  1. 首先在栈里面开辟main空间
  2. 在执行Person p1 = new…时，在堆里面有对象产生，p1指向 堆里面的方法
  3. 然后执行 int res = p1.getSum(1,2)时，在栈里面开辟getSum，通过地址把1，2传给他
  4. 再执行代码，return返回给res，getSum的空间释放

注意事项和细节：

  1. 同一个类中的方法调用：直接调用即可
  2. 如果跨类，对象名.方法名(参数)；

成员方法传参机制

  1. 基本数据类型，传递的是值（值拷贝），形参的任何改变不影响实参；
  2. 引用类型传递的是地址（传递也是值，但是值是地址），可以通过形参影响实参；
  3. 若是对象，则传的是地址。
